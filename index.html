Stage 2 — Identity Fingerprinting
ML model scores each listing against the user's verified profile using fuzzy name matching (Levenshtein), city/state validation, age-range check, and relative name cross-reference. Minimum 82% confidence threshold prevents wrongful removals.
fuzz.jsML scoring82% threshold
function matchIdentity(listing, user) { const nameScore = fuzz.tokenSetRatio(listing.name, user.fullName); const cityMatch = listing.city?.toLowerCase() === user.city?.toLowerCase(); const ageOk = Math.abs(listing.age - user.age) <= 3; const relMatch = listing.relatives?.some(r => user.knownRelatives.includes(r)); const score = (nameScore * 0.5) + (cityMatch ? 20 : 0) + (ageOk ? 20 : 0) + (relMatch ? 10 : 0); return { match: score >= 82, confidence: score }; }

PII Security Architecture
Encryption at Rest
All PII columns use AES-256-GCM. Each user gets a unique Data Encryption Key (DEK) generated by AWS KMS. DEKs are envelope-encrypted with a KMS CMK — never stored in plaintext. Key rotation: CMK rotated annually, DEKs re-encrypted on plan change or user request. Email is also SHA-256 hashed (email_hash) for lookups — enabling queries without decrypting.
Data in Transit
TLS 1.3 enforced end-to-end. HSTS with 1-year max-age. Certificate pinning in mobile apps. Internal service mesh uses mTLS (AWS App Mesh). No PII in URLs, query strings, or logs. CloudFront + WAF sits in front of all public endpoints.
Access Control
Row-Level Security (RLS) on PostgreSQL — users can only query their own rows. Service accounts use IAM roles with least-privilege. No direct DB access from Lambda/EC2 — all queries via parameterized prepared statements. Secrets Manager rotates DB credentials every 30 days.
Deletion & Retention
GDPR Art. 17 / CCPA §1798.105 deletion: soft-delete sets deleted_at, then hard-delete + KMS DEK destruction within 30 days. Audit log retained 7 years (CCPA compliance), but contains NO PII — only hashed IDs and event types. Removal certificates in S3 purged after 7 years with lifecycle policy.

Federal & International Framework
CCPA / CPRA — California Consumer Privacy Act
Primary U.S. framework. §1798.105 grants the right to deletion. Brokers with >$25M revenue or handling 100K+ consumers must comply within 45 days. Penalties: $2,500 unintentional / $7,500 intentional per violation. GeoSent files §1798.105 deletion requests with citation for all California-regulated brokers.
GDPR Article 17 — Right to Erasure (EU/UK)
Controllers must respond within 30 days. Fines up to €20M or 4% global annual revenue. Applies to any U.S. broker processing EU resident data. Used for ~12% of GeoSent removals for international users.
CAN-SPAM Act — Email Marketing Opt-Out
FTC-enforced. Brokers must process opt-out requests within 10 business days. Penalties: $51,744 per email per violation. GeoSent auto-generates CAN-SPAM opt-outs for all email list brokers.
